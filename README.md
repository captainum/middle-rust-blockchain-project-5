# Проектная работа модуля 5. Поиск ошибок и оптимизация

## Краткое описание

Доработки в части функционала (исправления ошибок логики, взаимодействия с памятью, а также оптимизации) описаны
в рамках docstring к функциям.

Были добавлены недостающие тесты, что привело к полному покрытию функционала при тестировании.

Директория artifacts содержит в себе листинги проверок сборки, тестирования, взаимодействия с памятью 
начального состояния (`*_initial.txt`), после исправлений логики и проблем работы с памятью (`*_first_iteration.txt`),
а также после оптимизационных доработок (`*_final.txt`).

При поиске проблем с памятью использовались `miri` и `valgrind`. Так как целевой системой, на которой выполнялась 
работа, является MacOS26, то появилась необходимость в использовании `valgrind` в докеризированном формате, для чего
был подготовлен `Dockerfile`. (По итогу улучшений `valgrind` не показывал ошибок в самих функциях, которые были 
доработаны, но остались ошибки в самом rustup, из листинга были убраны, т.к. не имеют отношения к написанному
пользовательскому коду). Были произведены запуски тестов под санитайзерами, `address` ошибок не обнаружил, 
`thread` собрать не получилось (и на целевой системе, и в докере), но в целом возможные проблемы с многопоточностью 
также покрыты запуском тестов под `valgrind` с использованием инструмента `helgrind`. 

Для удобства запуска тестирования описанными инструментами подготовлен скрипт `scripts/full_tests.sh`.

Предоставлен `flamegraph.svg`, построенный на основе запуска `demo.rs`, на котором видны проблемные в части 
производительности участки кода. Проблемными функциями оказались функции из `algo.rs` (постоянная сортировка, 
рекурсивные вызовы). Они же в дальнейшем и улучшались с точки зрения производительности. 
Описания улучшений приведены в docstring функций.

Измерения бенчмарков приведены в качестве HTML-отчета в `artifacts/criterion/reports/index.html`.
В `artifacts/criterion/reports/slow_dedup_broken/history.html` и `artifacts/criterion/reports/slow_fib_broken/history.html` 
можно увидеть прирост производительности.

Для удобства запуска профилирования подготовлен скрипт `scripts/profile.sh`.

Изменения и доработки фиксировались в истории коммитов.

Базовые проверки добавлены в CI.

## Список команд

### Проверка сборки (без компиляции)

```
cargo check
```

### Запуск тестов

```
cargo test
```

### Запуск тестов с miri

```
cargo +nightly miri test
```

### Проверка работоспособности под valgrind для поиска утечек / ошибок памяти

```
docker build -t rust-valgrind-tests . \
docker run --rm -v $(pwd):/app rust-valgrind-tests
```

или

```
./scripts/valgrind.sh
```

### Запуск полного цикла тестирования

```
./scripts/full_tests.sh
```

### Запуск профилирования

```
cargo criterion --bench criterion
```

или

```
./scripts/profile.sh
```

### Формирования отчета о покрытии

```
cargo llvm-cov --html
```
